# 🚀 Working Backwards – Stage 1 (고객에서 시작)
## 디지털 독서·기록·공유 플랫폼 (가칭: habitree reading hub)

---

## 1) 누가 고객인가? (Who is the Customer?)

우리 서비스는 3가지 핵심 고객군을 대상으로 한다.  
그러나 WB 원칙에 따라, 메인 페르소나는 개인 독자이며 나머지는 확장 가능한 서브 페르소나이다.

### 🎯 메인 고객 (Primary Persona)

**꾸준히 책을 읽고 기록하려는 개인 독자**

- 독서 습관을 유지하고 싶지만 기록이 흩어져 정리가 어렵다
- 인상 깊은 문장을 다시 찾고 싶지만 어디에 저장했는지 기억나지 않는다
- 기록을 남기고 공유하고 싶은 욕구가 있다
- 자신의 독서 히스토리와 성장을 눈으로 확인하고 싶다

### 🟦 서브 고객군 A: 독서모임 운영자

- 모임 구성원들의 읽기 현황·필사·인증 데이터를 한 곳에서 보고 싶다
- 운영 부담을 줄이고 모임의 질을 높이는 도구가 필요하다

### 🟦 서브 고객군 B: 북튜버·지식 콘텐츠 크리에이터

- 구독자와 함께 읽기 활동을 운영하고 싶다
- 여러 플랫폼(인스타/유튜브/디스코드)에 분산된 소통·기록을 통합하고 싶다
- 커뮤니티 흐름을 관리하고 팬덤을 쌓고 싶다

## 2) 고객의 문제나 기회는 무엇인가? (Problem / Opportunity)

### 🔥 A. 공통 핵심 문제 (Core Pain Points)

모든 고객군이 공통으로 겪는 문제는 아래 4개다.

- 기록이 여러 앱(카톡/사진/노션/메모)에 흩어져 있다
- 기록을 다시 찾기 어렵다
- 개인·그룹의 독서 흐름과 성장 히스토리가 남지 않는다
- 기록/참여 데이터가 분산되어 정리·공유가 어렵다

### 🔥 B. 개인 독자의 고유 문제

- 특정 문장·구절을 다시 찾고 싶은데 불가능하다
- 감동을 기록해도 결국 잊혀진다
- SNS나 모임에서 공유하고 싶지만 정리가 안되어 포기하게 된다
- 기록이 지속되지 않으니 독서 습관이 끊긴다

### 🔥 C. 독서모임 운영자의 고유 문제

- 구성원의 진행 상황·참여도를 한눈에 파악 불가
- 공지/인증/필사/피드백이 도구별로 분산
- 운영 피로와 시간적 부담
- 데이터 기반 운영(보고서/리포트)이 불가능

### 🔥 D. 크리에이터의 고유 문제

- 구독자 참여 데이터를 수집하기 어렵다
- 함께 읽기 챌린지를 운영하고 싶은데 시스템이 없다
- 플랫폼마다 기록이 분리되어 커뮤니티 흐름이 약해진다
- 브랜딩/팬덤 강화의 기회를 놓치고 있다

## 3) 가장 중요한 고객 혜택은 무엇인가? (Key Customer Benefit)

### 📌 단 하나의 핵심 혜택

**"기록을 다시 찾고, 정리하고, 공유할 수 있는 경험 제공"**

이 핵심 혜택은 아래 3가지 서브 가치로 분해된다.

### ✔ 1) 기록이 흩어지지 않고 통합된다

- 사진, 필사, 메모, 인상 깊은 문장 등
- → 하나의 책/문서에 자동 연결된 DB로 저장

### ✔ 2) 언제든 다시 찾아볼 수 있다

- "내가 읽은 문장", "각 책의 핵심 메모", "연간 기록 흐름"
- → 검색/필터링/타임라인으로 즉시 검색 가능

### ✔ 3) 내 기록을 쉽게 공유할 수 있다

- 독서모임 공유
- SNS 카드 뉴스
- 블로그 공유용 인용구
- → 공유 기능 자체가 고객의 '인정 욕구'를 만족시키고 습관 유지에 기여

> **이 3가지 혜택 = 플랫폼의 핵심 가치 제안 (Value Proposition)**

## 4) 고객이 원하는 것을 어떻게 알 수 있는가? (How do you know what customers want?)

아마존 WB 문서 구조에 맞춰 아래 4가지 기반을 사용한다.

### ① 실제 행동 데이터 기반

- 현재 책모임·습관모임 베타 테스트 참여자들의 인증 방식, 기록 방식, 참여율, 페이지 기록 데이터 수집
- 사람들이 카톡/노션/사진 등에 기록을 쌓는 패턴 분석
- → "흩어진 기록 → 찾지 못함 → 습관 끊김"이라는 데이터 패턴이 반복됨

### ② 설문 + 인터뷰 데이터 기반

- 독서모임 구성원 및 개인 독자 대상 설문
  - "책에서 본 문장을 다시 찾고 싶지만 못 찾은 경험이 있는가?"
  - "기록이 흩어져 관리 어려움을 느낀 적 있는가?"
- → 공통 Pain Point 도출

### ③ 프로토타입(더미 화면) 반응 테스트

- 개인 기록 모음, 필사 정리, 그룹 대시보드 화면 제시
- 기능 선호도 및 니즈 직접 확인
- → 가장 반응이 좋았던 화면 = "내가 기록한 문장 모아보기"

### ④ 본인의 경험 (Creator Empathy)에서 출발한 검증 포인트

- 너가 경험한 "기록 찾기 불가 / 공유의 어려움" 문제는 고객이 실제로 매우 크게 느끼는 니즈와 완전 일치
- → 강력한 1st-hand insight

## 5) 고객 경험은 어떤 모습인가? (Customer Experience)

아마존 WB 스타일로 "고객의 1주일 사용 흐름"으로 묘사한다.

### 🧭 📅 Day 1 — 가입 & 첫 기록

- 카카오/구글로 빠른 가입
- "올해 읽은 책 수 목표" 설정
- 첫 책 추가 (ISBN 검색 또는 촬영)
- 요약/필사/사진을 자동으로 한 곳에 저장

**고객 느낌 →**  
"아, 이제 내가 저장한 문장이 사라지지 않겠구나."

### 🧭 📅 Day 2 — 기록 자동 정리 경험

- 기존에 찍어둔 사진 불러오기
- 자동으로 "책/페이지/문장"으로 묶임
- 내 필사/사진이 책 내부에서 정렬됨

**고객 느낌 →**  
"내 기록이 하나의 책처럼 정돈된다."

### 🧭 📅 Day 3 — 공유 경험

- 인상 깊은 문장 선택 → 자동 카드뉴스 변환
- SNS 또는 모임방에 공유
- "와 이 문장 좋다!" 반응 받음

**고객 느낌 →**  
"내 기록을 공유하는 게 즐겁다."

### 🧭 📅 Day 5 — 모임/챌린지 참여

- 독서모임 리더가 만든 그룹에 참여
- 오늘 읽은 분량 체크
- 나의 필사/메모가 그룹 대시보드에 반영

### 🧭 📅 Day 7 — 누적 기록을 보는 첫 경험

- 이번 주 읽은 시간/페이지 확인
- 내가 저장한 필사/문장/메모 리스트가 타임라인 형태로 정리
- PDF/이미지로 내보내기 가능

**고객 느낌 →**  
"드디어 내 독서가 한눈에 보인다. 이건 계속하고 싶다."
