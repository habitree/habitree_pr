# Working Backwards & PRFAQ 프로세스 가이드

## 아마존의 핵심 철학

**"Earth's most customer-centric company"** - 세상에서 가장 고객 중심적인 회사

### Working Backwards 원칙

- 제품이나 서비스를 개발하기 전에 **고객의 관점에서 시작**
- 내부 구조나 재정적 목표가 아닌 **순수하게 고객 중심으로 사고**
- **"고객이 WOW라고 말하게 만드는 것"**이 목표
- 미래를 상상하며 아직 존재하지 않는 제품/서비스의 고객 경험을 설계

---

## WB 프로세스 단계

Working Backwards 프로세스는 다음 순서로 진행됩니다:

```
고객 → PR → FAQ → Visuals → 런칭
```

### ❌ 잘못된 접근
서비스/프로덕트 런칭 → PR (뉴스기사)

### ✅ 올바른 접근
고객 → PR → FAQ → Visuals → 런칭

---

## 각 단계별 상세 가이드

### 1단계: Customer (고객 정의)

**목표:** 고객을 명확히 정의하고, 그들의 문제와 니즈를 파악

**작성 항목:**
- 누가 고객인가? (Who is the Customer?)
  - 메인 페르소나 정의
  - 서브 페르소나 정의
- 고객의 문제나 기회는 무엇인가? (Problem / Opportunity)
  - 공통 핵심 문제
  - 페르소나별 고유 문제
- 가장 중요한 고객 혜택은 무엇인가? (Key Customer Benefit)
- 고객이 원하는 것을 어떻게 알 수 있는가? (How do you know what customers want?)
  - 실제 행동 데이터
  - 설문 + 인터뷰 데이터
  - 프로토타입 반응 테스트
  - 본인의 경험 (Creator Empathy)

**체크리스트:**
- [ ] 메인 페르소나가 명확히 정의되었는가?
- [ ] 고객의 핵심 문제가 구체적으로 기술되었는가?
- [ ] 고객 혜택이 측정 가능한가?
- [ ] 고객 니즈에 대한 검증 근거가 있는가?

---

### 2단계: Press Release (보도자료)

**목표:** 고객 관점에서 작성된 제품 출시 보도자료

**작성 원칙:**
- 고객이 읽고 흥미를 느끼도록 작성
- 제품의 주요 특징과 고객 혜택을 강조
- 기술적 세부사항보다는 고객 가치에 집중
- 미래의 고객 경험을 생생하게 묘사

**구조:**
1. 헤드라인 (고객이 주목할 만한 제목)
2. 서브헤드라인 (핵심 가치 제안)
3. 문제 정의 (고객이 겪는 문제)
4. 솔루션 소개 (제품이 어떻게 문제를 해결하는가)
5. 주요 기능 및 혜택
6. 고객 인용 (가상의 고객 후기)
7. 출시 정보

**체크리스트:**
- [ ] 고객 관점에서 작성되었는가?
- [ ] 핵심 가치 제안이 명확한가?
- [ ] 고객이 WOW라고 말할 만한 내용인가?
- [ ] 기술적 용어 없이 이해하기 쉬운가?

---

### 3단계: FAQ (자주 묻는 질문)

**목표:** 제품 개발 과정에서 예상되는 내부 및 외부의 질문과 답변 정리

**구성:**
1. **Customer FAQ (고객 FAQ)**
   - 고객이 제품에 대해 가질 수 있는 질문
   - 사용법, 기능, 가격, 보안 등
   
2. **Stakeholder FAQ (이해관계자 FAQ)**
   - 경영진, 투자자, 개발팀 등이 가질 수 있는 질문
   - 비즈니스 모델, 기술적 구현, 경쟁 우위, 리스크 등

**작성 원칙:**
- 예상되는 모든 질문을 미리 생각
- 솔직하고 구체적인 답변 제공
- 불확실한 부분은 명시
- 각 답변이 검증 가능한 근거를 포함

**체크리스트:**
- [ ] 고객 FAQ가 충분히 작성되었는가?
- [ ] 이해관계자 FAQ가 충분히 작성되었는가?
- [ ] 모든 답변이 구체적이고 검증 가능한가?
- [ ] 리스크와 대응 전략이 명시되었는가?

---

### 4단계: Customer Experience (고객 경험)

**목표:** 고객의 실제 사용 여정과 경험을 상세히 설계

**작성 항목:**
- 고객 여정 (Customer Journey)
  - 첫 접점부터 장기 사용까지의 흐름
  - 주요 터치포인트
  - 감정 곡선
- 주요 사용 시나리오
  - Day 1, Day 2, Day 7 등 시간별 경험
  - 고객의 느낌과 반응
- WOW 모멘트 정의
  - 고객이 감동받을 만한 순간
  - 경쟁사와 차별화되는 경험

**체크리스트:**
- [ ] 고객 여정이 단계별로 명확히 정의되었는가?
- [ ] 주요 터치포인트가 모두 포함되었는가?
- [ ] WOW 모멘트가 구체적으로 기술되었는가?
- [ ] 고객의 감정 변화가 반영되었는가?

---

### 5단계: Problem Statement (문제 정의)

**목표:** 해결하려는 문제를 명확히 정의하고 가설을 수립

**작성 항목:**
1. Customer (고객)
   - 고객의 현재 행동과 상황
2. Core Problem (핵심 문제)
   - 해결해야 할 문제의 본질
   - 현재 솔루션의 한계
3. Opportunity (기회)
   - 시장 공백
   - 해결 가능성
4. Hypothesis Set (가설 세트)
   - Problem Hypothesis
   - Solution Hypothesis
   - Behavior Hypothesis
   - Value Hypothesis
   - Business Hypothesis

**체크리스트:**
- [ ] 문제가 구체적이고 측정 가능한가?
- [ ] 가설이 검증 가능한가?
- [ ] 기회가 명확히 정의되었는가?
- [ ] 비즈니스 가설이 현실적인가?

---

## 프로세스 활용 팁

1. **고객 중심 사고 유지**
   - 모든 단계에서 "고객이 원하는 것은 무엇인가?"를 먼저 물어보기
   - 내부 편의보다 고객 경험 우선

2. **반복적 개선**
   - 각 단계를 완료한 후 다시 검토
   - 고객 피드백을 반영하여 문서 수정

3. **팀과 공유**
   - 모든 이해관계자와 문서 공유
   - 팀 전체가 같은 비전을 공유하도록

4. **검증 중심**
   - 모든 가설은 검증 가능해야 함
   - 데이터와 근거 기반으로 작성
