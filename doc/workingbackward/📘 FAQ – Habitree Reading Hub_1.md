## 🔵 I. Customer FAQ (고객 FAQ)

### 1. Habitree는 무엇인가요?

Habitree는 책을 읽으며 남긴 필사, 사진, 메모, 인상 깊은 문장을 자동으로 정리하고 다시 찾게 해주는 책 전용 기록·공유 플랫폼입니다.

### 2. 왜 Habitree가 필요한가요? 기존 앱으로도 기록 가능한데요?

기록은 어디서나 가능하지만 문제는 다음과 같습니다:

- 기록이 여러 앱에 흩어짐
- 다시 찾기 어려움
- 책 단위로 정리되지 않음
- 공유가 불편함

Habitree는 책 하나를 위한 전용 저장소로 기록을 자동으로 한 곳에 정리해줍니다.

### 3. 기록은 어떻게 자동 정리되나요?

- 필사 입력
- 사진 업로드
- 텍스트 캡처
- 메모 작성

모든 것이 해당 책의 노트로 자동 연결됩니다.

### 4. 인상 깊었던 문장을 다시 찾을 수 있나요?

네. Habitree는 기록을 문장 단위로 저장하며 책 제목·날짜·주제·필사 유형 등으로 빠르게 검색할 수 있습니다.

### 5. 다른 사람의 필사나 생각도 볼 수 있나요?

네, 원할 경우 공개된 필사·메모를 다른 사용자도 볼 수 있습니다.  
기본값은 비공개이며 사용자 선택에 따라 공개 여부를 결정합니다.

### 6. 독서모임 기능이 있나요?

있습니다.

- 읽는 책 리스트 공유
- 구성원 필사/메모 공유
- 진행 현황 표시
- 모임 활동 요약 제공

모임 운영자에게 특히 유용합니다.

### 7. SNS나 블로그에 쉽게 공유할 수 있나요?

네. 필사·문장·메모를 카드뉴스 형태로 자동 생성하여 인스타그램, 카카오톡, 블로그 등에 바로 공유할 수 있습니다.

### 8. 내 기록은 어떻게 보호되나요?

모든 기록은 암호화된 DB에 저장되며 기본 비공개 상태입니다.  
공개 여부는 아래에서 선택할 수 있습니다:

- 완전 비공개
- 그룹에만 공개
- 전체 공개

### 9. 가격은 어떻게 되나요?

초기 MVP는 무료입니다.  
추후 구독 모델에는 아래와 같은 기능이 포함될 수 있습니다:

- 고급 필사 검색
- 장기 독서 리포트
- 회고·성장 분석
- 모임 운영 대시보드
- 크리에이터 커뮤니티 기능

### 10. 책 대여/판매/선물 기능도 있나요?

초기에는 제공되지 않습니다.  
하지만 Habitree 내에서 신뢰 네트워크가 형성되면 아래 기능들이 2차 단계 BM으로 확장됩니다:

- 사용자 간 책 대여
- 중고책 판매
- 책 선물하기

### 11. 누구에게 추천하나요?

- 필사·메모를 남기고 싶은 독자
- 읽었던 문장을 다시 찾고 싶은 사람
- 독서모임 운영자/참여자
- 책 문장을 SNS로 공유하는 사람
- 지식 콘텐츠를 만드는 크리에이터

## 🟣 II. Stakeholder FAQ (이해관계자 FAQ)

### 1. 이 문제는 해결할 가치가 있는 문제인가?

그렇다. 독자들은 오랫동안 "기록을 다시 찾지 못하는 문제"를 반복 경험해왔다.  
이 문제는 실질적·감정적 Pain Point이며 현재 어떤 도구도 책 전용 정리·검색 경험을 제공하지 못하고 있다.

### 2. 경쟁 서비스와의 차별점은 무엇인가?

#### 노션

- 범용 도구이며 책 기반 구조는 아님
- 템플릿 제작 난이도 ↑
- 자동 정리/검색 기능 없음

#### 플라이북, 리디 하이라이트 등

- 기능 단편적
- 필사/메모/사진 통합 관리 불가

#### 블로그·SNS

- 기록은 누적되지만 정리·검색 불가
- 노이즈 많음, 정보 질 낮음

#### Habitree의 차별성

→ **책 기반 DB + 자동 정리 + 검색 + 공유 + 커뮤니티 결합**  
이 부분이 독보적이다.

### 3. 고객에게 WOW 경험을 주는 핵심 기능은?

- 기록 자동 정리 (Zero-Effort Logging)
- 문장 재발견 (Search & Recall)
- 독서 타임라인 시각화
- 즉시 공유 (One-Tap Sharing)

### 4. 초기 성공 지표(Metrics)은 무엇인가?

- 활성 사용자 수 (WAU)
- 기록 재사용률 (검색)
- 필사·메모 작성수
- 타임라인 조회 빈도
- 공유 횟수
- 독서모임 참여율

### 5. 기술적으로 구현 가능하나요?

네. MVP는 아래 기술로 충분히 가능:

- React
- Firebase Authentication
- Firestore Database
- 이미지 OCR
- 책 정보 API
- 서버리스 기반 운영 (저비용)

### 6. 프라이버시 위험은 없나요?

없습니다.

- 기본 비공개
- 사용자가 선택한 기록만 공개
- Firestore 보안 규칙으로 사용자별 접근 철저히 분리

### 7. 왜 책 거래 기능은 초기 론칭에서 제외되었나요?

실물 거래는 다음과 같은 높은 복잡성을 포함합니다:

- 물류
- 정산
- 사기 방지
- 신뢰 기반 시스템

따라서 사용자 기반과 신뢰 구조가 쌓인 후 2차 확장 BM으로 도입될 예정입니다.

### 8. 실패 가능성과 대응 전략은 무엇인가요?

#### 가능한 리스크

- 자동화 품질 낮음
- 재방문 동기 부족
- 공유 기능의 매력 부족
- UI/UX 복잡성

#### 대응 전략

- 검색 경험 강화
- 공유 템플릿 지속 개선
- '나의 독서 리포트'로 감정적 동기 부여
- 모임 기능 강화로 커뮤니티 유지

### 9. 장기 성장 전략은 무엇인가?

- 개인 기록 경험 중심의 사용자 확보
- 독서모임을 통한 바이럴
- 크리에이터 커뮤니티 형성
- 공유 문화 중심 확산
- 기록 기반 추천 알고리즘 개발
- Book Circulation 모델 확장
