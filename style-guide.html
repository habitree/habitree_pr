1. 디자인 컨셉

Habitree 메인 페이지의 핵심 디자인 원칙과 분위기.

컨셉 키워드

Clean & Impactful — 깔끔하지만 강렬한 인상. 불필요한 장식 없이 콘텐츠 자체가 돋보이는 설계.

  • Dark-to-Light 전환: Hero/Story는 다크 모드, 서비스/Quiz는 라이트 모드. 자연스러운 그라데이션으로 연결
  • Single-page narrative: 스크롤 하나로 문제 → 해답 → 서비스 → 빌더 스토리 → 로드맵까지 자연스러운 내러티브 흐름
  • Minimal animation: 스크롤 기반 reveal과 카운터 정도만 사용. GPU 가속 속성(transform, opacity)만 애니메이션
  • 서비스별 컬러 존: 각 서비스 섹션은 고유 배경색+컬러 체계로 시각적 분리
  • Frameworkless: 순수 HTML/CSS/JS만 사용. 빌드 도구 없이 정적 파일로 동작

페이지 구조 흐름

Dark Zone     ┃  Hero (전체화면, 브랜드 카드)
              ┃  Problem (개발자/독서가 페인 포인트)
              ┃
Transition    ┃  Answer (다크→라이트 그라데이션, 인용구)
              ┃
Light Zone    ┃  Stats (핵심 수치 3개)
              ┃  Linkmap (서비스 01, 블루 배경)
              ┃  ReadTree (서비스 02, 크래프트 배경)
              ┃  Quiz (사용자 타입 선택)
              ┃
Dark Zone     ┃  Behind (빌더 스토리, 터미널 UI)
              ┃
Light Zone    ┃  Next (로드맵 3컬럼)
              ┃
Dark Zone     ┃  Footer (메가 타이포 + 링크 그리드)

컬러 적용 규칙

  • 다크/라이트 전환은 섹션 단위로 적용. 한 섹션 안에서 혼용하지 않음
  • 서비스 섹션은 고유 배경색 + 고유 primary로 브랜드 구분
  • Accent Green은 CTA, 하이라이트, 진행 상태에 사용
  • Accent Gold는 강조, 보조 하이라이트에 사용
  • dim 버전(hsla(..., 0.08~0.1))은 배지/태그 배경에 사용

3. 타이포그래피

3종 폰트 시스템: UI / 헤드라인 / 코드.

보이지 않는 습관을 보이는 자산으로

쌓이지 않으면 자산이 아닙니다
Noto Serif KR — 헤드라인, 인용구, 감성 강조
AES-256-GCM // 1,800+ commits
JetBrains Mono — 코드, 기술 수치, 라벨, 터미널

적용 규칙

  • H1은 letter-spacing: -0.03em, H2는 -0.02em (타이트한 자간)
  • body line-height: 1.65, 본문 단락 line-height: 1.7~2.0
  • 라벨/배지: text-transform: uppercase; letter-spacing: 0.15em
  • clamp()로 모바일~데스크톱 유동 스케일 적용

7. 애니메이션

성능 중심의 최소 애니메이션 시스템.

Reveal (스크롤 진입)

IntersectionObserver로 뷰포트 진입 감지. .reveal 클래스에 .visible 추가.

/* 초기 상태 */
.reveal {
  opacity: 0;
  transform: translateY(28px);
  transition: opacity 0.7s var(--ease), transform 0.7s var(--ease);
}
/* 뷰포트 진입 */
.reveal.visible {
  opacity: 1;
  transform: none;
}

/* Stagger (자식 순차 등장) */
.stagger > * { transition-delay: calc(var(--i, 0) * 0.08s); }
/* 실제: nth-child로 0s, 0.08s, 0.16s, 0.24s, 0.32s */

Split Text (히어로 글자 애니메이션)

헤드라인의 각 글자를 <span>으로 분리, 0.035s 간격으로 순차 등장.

/* JS가 각 글자에 인라인 스타일 적용 */
span {
  display: inline-block;
  opacity: 0;
  transform: translateY(24px);
  transition: opacity 0.5s var(--ease), transform 0.5s var(--ease);
  transition-delay: calc(index * 0.035s);
}

Counter (숫자 카운트업)

data-target 속성의 목표값까지 1.2초간 ease-out 카운트.

// cubic ease-out: (1 - Math.pow(1 - progress, 3)) * target
// IntersectionObserver threshold: 0.5 (절반 보일 때 시작)

Ticker / Marquee (무한 스크롤)

@keyframes ticker { to { transform: translate3d(-50%, 0, 0); } }
.ticker-track { animation: ticker 40s linear infinite; }
.marquee-track { animation: ticker 35s linear infinite; }

Hover 패턴

translateY(-2px) 버튼
translateY(-4px) + shadow 카드
translateX(6px) 피처
/* 버튼: 살짝 떠오름 */
.btn:hover { transform: translateY(-2px); }

/* 카드: 떠오름 + 그림자 */

/* 피처 아이템: 오른쪽 이동 */
.feature-item:hover { transform: translateX(6px); }

/* 이미지: 미세 확대 */
.showcase-visual:hover { transform: scale(1.02); }

접근성: prefers-reduced-motion

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
  .reveal { opacity: 1 !important; transform: none !important; }
  .ticker-track, .marquee-track { animation: none !important; }
}

8. 섹션 패턴

각 섹션의 설계 패턴과 적용 방법.

Hero 패턴

  • min-height: 100vh 풀스크린 + flexbox 수직 중앙
  • 좌: 텍스트(H1 + 서브 + CTA) / 우: 브랜드 카드 — 2컬럼 그리드
  • 배경: 미세한 radial-gradient glow (hsla(155, 60%, 40%, 0.06))
  • 하단: 키워드 무한 스크롤 ticker

서비스 쇼케이스 패턴

  • 2컬럼: 텍스트(라벨→로고→태그라인→설명→기능4개→태그→CTA) + 스크린샷
  • Linkmap: 텍스트 좌, 이미지 우 / ReadTree: 이미지 좌, 텍스트 우 (교차 배치)
  • 섹션별 고유 배경색으로 시각적 분리
  • 서비스 라벨: font-mono, 0.6rem, uppercase, letter-spacing 0.15em

전환 섹션 (Answer)

  • background: linear-gradient(to bottom, var(--dark-bg), var(--light-bg))
  • 스크롤 기반 단어별 fade-in 효과
  • 중앙 정렬, serif 폰트, 최대 너비 600px

Footer 패턴

  • 상단: 메가 타이포 ("Habitree" 4~6% 불투명도로 배경 느낌)
  • 중간: 4컬럼 그리드 (브랜드 설명 2fr + 링크 3개 1fr씩)
  • 하단: 보더 + 카피라이트 + 소셜

9. 반응형

3단계 브레이크포인트.

브레이크포인트변경 사항
> 768px기본 데스크톱 레이아웃
≤ 768px모든 그리드 1컬럼, 네비 ul 숨김, 히어로 비주얼 상단 이동, 푸터 2컬럼
≤ 480pxcontainer padding 16px, CTA 버튼 100% 너비, 푸터 1컬럼
 
@media (max-width: 768px) {
  .hero-inner { grid-template-columns: 1fr; text-align: center; }
  .hero-visual { order: -1; }       /* 이미지 먼저 */
  .showcase-split { grid-template-columns: 1fr !important; }
  .fixed-nav ul { display: none; }  /* 네비 링크 숨김 */
}

@media (max-width: 480px) {
  .container { padding: 0 16px; }
  .hero-ctas .btn { width: 100%; justify-content: center; }
}
